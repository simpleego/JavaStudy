# 자바 개발 환경 설정

## 개발 환경 설정 순서

1.  JDK 설치 (Azul Zulu JDK 17 )설치 및
2.  자바 환경 변수 설정
3.  메모장으로 자바 소스 코드 작성
4.  `javac` 명령어로 컴파일하여 바이트코드(`.class` 파일) 생성
5.  `java` 명령어로 프로그램 실행

-----

## Ⅰ. 자바 개발 환경(JDK) 설정

### 1단계: Azul Zulu JDK 17 다운로드 및 설치

1.  **다운로드:** Azul Zulu 공식 웹사이트에서 **Zulu 17 LTS 버전**을 선택하고, 윈도우용 \*\*MSI Installer (64-bit)\*\*를 다운로드합니다.
2.  **설치:** 다운로드한 설치 파일(`.msi`)을 실행합니다.
      * 설치 과정 중 특별한 설정 변경 없이 **Next**를 누르며 진행합니다.
      * **⚠️ 중요:** 설치 옵션 중 **"Set JAVA\_HOME variable"** 옵션이 있다면 **반드시 활성화**하고 진행합니다. (Zulu JDK는 이 옵션을 제공하는 경우가 많아 편리합니다.)
3.  **설치 경로 확인:** 기본적으로 `C:\Program Files\Zulu\zulu-17`와 유사한 경로에 설치됩니다. 이 경로를 기억해 둡니다.

### 2단계: 환경 변수 설정 (수동 설정 시)

설치 시 `JAVA_HOME` 설정을 자동으로 진행했다면 이 단계를 건너뛰어도 되지만, 수업에서는 **환경 변수의 원리 이해**를 위해 수동 설정을 진행하는 것이 좋습니다.

#### 2.1. `JAVA_HOME` 환경 변수 설정

`JAVA_HOME`은 자바 개발 도구(JDK)의 루트 경로를 지정하는 시스템 변수입니다.

1.  \*\*[윈도우 검색]\*\*에서 \*\*"시스템 환경 변수 편집"\*\*을 검색하여 실행합니다.
2.  **[시스템 속성]** 창에서 **[환경 변수(N)...]** 버튼을 클릭합니다.
3.  **[시스템 변수]** 섹션에서 \*\*[새로 만들기(W)...]\*\*를 클릭합니다.
      * **변수 이름:** `JAVA_HOME`
      * **변수 값:** 설치된 JDK 17의 루트 디렉터리 경로 (예: `C:\Program Files\Zulu\zulu-17`)
4.  \*\*[확인]\*\*을 눌러 저장합니다.

#### 2.2. `Path` 환경 변수 편집

`Path` 변수는 윈도우의 어떤 디렉터리에서든 `java`나 `javac` 명령어를 실행할 수 있도록 명령어의 위치를 지정해 줍니다.

1.  **[시스템 변수]** 목록에서 `Path` 변수를 찾아 선택하고 \*\*[편집(I)...]\*\*을 클릭합니다.
2.  \*\*[새로 만들기(N)]\*\*를 클릭하고 다음 값을 입력합니다. (이때, `%JAVA_HOME%` 변수를 활용하여 경로를 지정하는 것이 좋습니다.)
      * **추가할 값:** `%JAVA_HOME%\bin`
3.  모든 창에서 \*\*[확인]\*\*을 눌러 변경 사항을 저장합니다.

#### 2.3. 설정 확인

**명령 프롬프트** 또는 **Git Bash**를 새로 열고 다음 명령어를 입력하여 JDK 설치와 환경 변수 설정을 확인합니다.

```bash
java -version
javac -version
```

**출력 예시:**

```
java version "17.0.9" 2023-10-17 LTS
Java(TM) SE Runtime Environment (build 17.0.9+8-LTS)
Zulu Platform for SE (build 17.0.9+8-LTS)

javac 17.0.9
```

## Ⅱ. 메모장으로 자바 코드 작성

### 1단계: 소스 파일 생성

1.  코드를 저장할 새로운 디렉터리(예: `C:\JavaProject`)를 생성합니다.
2.  **메모장**을 엽니다.
3.  다음과 같은 간단한 코드를 작성합니다.

<!-- end list -->

```java
// 파일 이름: SimpleHello.java
public class SimpleHello {
    public static void main(String[] args) {
        System.out.println("Hello, Java World! - from Memo Pad");
    }
}
```

### 2단계: 파일 저장

1.  메모장의 \*\*[파일] \> [다른 이름으로 저장]\*\*을 선택합니다.
2.  저장 위치를 위에서 만든 디렉터리(`C:\JavaProject`)로 지정합니다.
3.  **⚠️ 중요:**
      * **파일 이름:** `public class` 뒤에 선언한 이름과 **정확히 동일**하게 `SimpleHello.java`로 입력합니다. (자바는 대소문자를 구분합니다.)
      * **파일 형식:** 반드시 \*\*"모든 파일(\*.\*)"\*\*로 변경하고 저장합니다. (기본값인 `.txt`로 저장되면 안 됩니다.)

## Ⅲ. 컴파일 및 실행

이제 Git Bash나 윈도우 명령 프롬프트를 사용하여 코드를 컴파일하고 실행합니다.

### 1단계: 컴파일 (Compiler: `javac`)

-----

1.  **명령 프롬프트/Git Bash**를 엽니다.

2.  작성한 소스 파일이 있는 디렉터리로 이동합니다.

    ```bash
    cd C:\JavaProject
    ```

3.  `javac` 명령어와 소스 파일 이름을 사용하여 컴파일을 수행합니다.

    ```bash
    javac SimpleHello.java
    ```

4.  **확인:** 컴파일에 성공하면, 같은 디렉터리에 **`SimpleHello.class`** 파일(자바 바이트코드)이 생성됩니다. 아무런 메시지가 출력되지 않으면 성공한 것입니다. (에러가 발생하면 오류 메시지가 출력됩니다.)

### 2단계: 실행 (Interpreter: `java`)

-----

1.  `java` 명령어와 클래스 이름(확장자 `.class` 제외)을 사용하여 프로그램을 실행합니다.

    ```bash
    java SimpleHello
    ```

2.  **결과 확인:** 화면에 다음 문구가 출력되는지 확인합니다.

    ```
    Hello, Java World! - from Memo Pad
    ```

이 과정을 통해 학생들은 자바 코드가 IDE의 도움 없이 어떻게 컴파일러(`javac`)를 통해 바이트코드가 되고, 최종적으로 JVM(`java`)을 통해 실행되는지 기본적인 원리를 이해할 수 있습니다.
