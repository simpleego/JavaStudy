# 자바(Java)의 자료형(Data Types)
> 변수에 저장될 데이터의 종류와 크기를 정의하는 데 필수적입니다.
> \*\*기본 자료형(Primitive Data Types)\*\*과 \*\*참조 자료형(Reference Data Types)\*\*으로 나뉩니다.
-----

## 💻 자바 자료형 정리 (Java Data Types)

### 1\. 기본 자료형 (Primitive Data Types)

기본 자료형은 실제 값(데이터)을 저장하며, 총 8가지가 있습니다. 모두 스택(Stack) 메모리 영역에 저장됩니다.

<img width="393" height="395" alt="image" src="https://github.com/user-attachments/assets/b8477cc0-e87f-4309-a547-dad3029dcb6a" />


| 분류 | 자료형 | 크기 (바이트) | 저장 가능한 값의 범위 (대략) | 예제 |
| :--- | :--- | :--- | :--- | :--- |
| **정수형** | `byte` | 1 | $-128$ \~ $127$ | `byte b = 10;` |
| | `short` | 2 | $-32,768$ \~ $32,767$ | `short s = 5000;` |
| | `int` | 4 | 약 $\pm 2$ 억 | `int i = 100000;` |
| | `long` | 8 | 매우 큰 정수 (L 또는 l 접미사 사용) | `long l = 10000000000L;` |
| **실수형** | `float` | 4 | 단정밀도 (F 또는 f 접미사 사용) | `float f = 3.14f;` |
| | `double` | 8 | 배정밀도 (가장 일반적인 실수형) | `double d = 3.141592;` |
| **문자형** | `char` | 2 | $0$ \~ $65,535$ (유니코드 문자 하나) | `char c = 'A';` |
| **논리형** | `boolean` | 1 | `true` 또는 `false` | `boolean isTrue = true;` |

> **💡 참고:** 자바에서 정수 리터럴은 기본적으로 `int`로 간주되며, 실수 리터럴은 기본적으로 `double`로 간주됩니다.

-----

### 2\. 참조 자료형 (Reference Data Types)

참조 자료형은 실제 값이 아닌 \*\*값이 저장된 메모리 주소(참조)\*\*를 저장합니다. 주소는 힙(Heap) 메모리 영역의 객체를 가리키며, 스택에는 해당 주소만 저장됩니다.

  * **종류:** 클래스(Class), 인터페이스(Interface), 배열(Array), 열거형(Enum) 등
  * **대표적인 예시:** `String`, `Scanner`, 사용자 정의 클래스 등

| 자료형 | 설명 | 예제 |
| :--- | :--- | :--- |
| **`String`** | 문자열을 다루는 가장 기본적인 클래스 | `String name = "Gemini";` |
| **배열** | 동일한 타입의 데이터를 연속적으로 저장하는 구조 | `int[] numbers = {1, 2, 3};` |
| **클래스** | 객체를 생성하는 틀 (사용자 정의 자료형) | `MyClass obj = new MyClass();` |

#### `String` 자료형 예제

`String`은 자바에서 매우 자주 사용되는 참조 자료형입니다.

```java
public class StringExample {
    public static void main(String[] args) {
        // String 객체 생성 (문자열 리터럴 사용)
        String greeting = "Hello, Java!"; 
        
        // 문자열 연결
        String message = greeting + " Welcome.";
        
        System.out.println(message); // 출력: Hello, Java! Welcome.
        
        // String은 객체이므로 메소드 사용 가능
        System.out.println("길이: " + message.length()); // 출력: 길이: 21
    }
}
```

## 📚 기본 자료형과 참조 자료형의 차이

이 둘의 가장 큰 차이점은 변수가 실제로 무엇을 저장하는지입니다.

| 구분 | 기본 자료형 (Primitive) | 참조 자료형 (Reference) |
| :--- | :--- | :--- |
| **저장 값** | 실제 데이터 값 | 객체의 메모리 주소 (참조) |
| **메모리 위치** | 스택(Stack) 영역 | 힙(Heap) 영역 (값) + 스택 영역 (주소) |
| **null 가능성** | 불가능 | 가능 (참조하는 객체가 없음) |
| **기본값** | 각 타입의 기본값 (0, 0.0, false, \\u0000) | `null` |
