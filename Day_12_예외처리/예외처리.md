**핵심 요약:**  
자바 예외처리는 프로그램 실행 중 발생하는 오류 상황을 안전하게 처리하기 위한 메커니즘입니다. 크게 **Checked Exception**과 **Unchecked Exception**으로 나뉘며, `try-catch-finally` 구문이나 `throws` 키워드를 통해 관리합니다.  

---

## 🧩 자바 예외처리 개념 정리

### 1. 예외(Exception)의 계층 구조
- **Throwable**  
  ┣ **Error** → 시스템 레벨 오류 (복구 불가)  
  ┗ **Exception** → 프로그램 레벨 오류 (복구 가능)

### 2. 예외의 종류
| 구분 | 설명 | 대표 예시 |
|------|------|-----------|
| **Checked Exception** | 반드시 처리해야 하는 예외. 컴파일 시점에 확인됨 | IOException, SQLException |
| **Unchecked Exception** | 선택적으로 처리 가능. 런타임 시점에 발생 | NullPointerException, IllegalArgumentException |

### 3. 처리 방법
- **try-catch-finally**: 예외 발생 시 catch 블록에서 처리, finally는 자원 정리용  
- **throws**: 메서드 선언부에서 예외를 호출한 쪽으로 전달  
- **throw**: 특정 예외를 직접 발생시킬 때 사용

## 💻 예제 코드

### Checked Exception 예제 (파일 읽기)
```java
import java.io.*;

public class FileReadExample {
    public static void main(String[] args) {
        try {
            BufferedReader br = new BufferedReader(new FileReader("test.txt"));
            String line = br.readLine();
            System.out.println("파일 내용: " + line);
            br.close();
        } catch (IOException e) {
            System.out.println("파일을 읽는 중 오류 발생: " + e.getMessage());
        } finally {
            System.out.println("자원 정리 완료");
        }
    }
}
```
- `IOException`은 **Checked Exception**이므로 반드시 `try-catch` 또는 `throws`로 처리해야 함.

---

### Unchecked Exception 예제 (나눗셈)
```java
public class DivisionExample {
    public static void main(String[] args) {
        try {
            int result = 10 / 0; // ArithmeticException 발생
            System.out.println("결과: " + result);
        } catch (ArithmeticException e) {
            System.out.println("0으로 나눌 수 없습니다: " + e.getMessage());
        }
    }
}
```
- `ArithmeticException`은 **Unchecked Exception**으로, 처리하지 않아도 컴파일은 되지만 런타임 오류 발생.

---

## ⚠️ 주의할 점
- **Checked Exception**은 반드시 처리해야 하므로 코드가 복잡해질 수 있음.  
- **Unchecked Exception**은 개발자가 직접 방어적 코드를 작성해야 함.  
- 예외를 무조건 잡기보다는 **의미 있는 처리**를 하는 것이 중요.

## 🚀 자바 예외 던지기(throw / throws)

### 1. `throw` 키워드
- 특정 시점에서 **직접 예외 객체를 발생**시킬 때 사용  
- 문법:  
  ```java
  throw new ExceptionType("예외 메시지");
  ```

### 2. `throws` 키워드
- 메서드 선언부에 붙여서, 해당 메서드가 **예외를 호출한 쪽으로 전달**할 수 있음을 명시  
- 문법:  
  ```java
  public void methodName() throws ExceptionType { ... }
  ```

### 3. 활용 방식
- **throw**: 실제 예외를 발생시킴  
- **throws**: 예외를 처리하지 않고 호출한 메서드로 넘김  
- Checked Exception은 반드시 `throws` 또는 `try-catch`로 처리해야 함  
- Unchecked Exception은 선택적으로 처리 가능

## 💻 예제 코드

### 사용자 정의 예외 만들기 + 던지기
```java
// 사용자 정의 예외 클래스
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class ThrowExample {
    // 나이 검증 메서드
    public static void checkAge(int age) throws InvalidAgeException {
        if (age < 18) {
            // 직접 예외 발생
            throw new InvalidAgeException("미성년자는 접근할 수 없습니다.");
        } else {
            System.out.println("접근 허용");
        }
    }

    public static void main(String[] args) {
        try {
            checkAge(15); // 예외 발생
        } catch (InvalidAgeException e) {
            System.out.println("예외 처리: " + e.getMessage());
        }
    }
}
```

### 실행 흐름
1. `checkAge(15)` 호출 → 조건 불만족 → `throw new InvalidAgeException(...)` 실행  
2. 예외가 발생했으므로 `throws`에 의해 호출한 쪽(`main`)으로 전달  
3. `try-catch` 블록에서 예외를 받아 처리  

---

## ⚡ 핵심 포인트
- `throw`: 예외 객체를 **실제로 던짐**  
- `throws`: 메서드가 예외를 **던질 수 있음을 선언**  
- **Checked Exception**은 반드시 처리해야 하므로 `throws` 선언이 자주 사용됨  
- **Unchecked Exception**은 필요할 때만 처리
