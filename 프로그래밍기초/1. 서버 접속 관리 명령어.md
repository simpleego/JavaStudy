# **서버 원격 접속 명령어**  
> 서버 원격 접속 및 관린 명령어는 
- **SSH(원격 접속)**
- **SCP(파일 복사)**
- **SFTP(파일 전송)**
 
> 이들은 모두 보안 연결을 기반으로 하며 서버 관리와 협업에 필수적입니다.

---

## 🔑 원격 접속 제어 기본 명령어

### 1. SSH (Secure Shell)
- **역할:** 원격 서버에 안전하게 접속하여 명령어 실행  
- **기본 사용법:**  
  ```bash
  ssh [옵션] 사용자명@호스트주소
  ```
- **주요 옵션:**
  - `-p [포트번호]` : 기본은 22번 포트, 다른 포트 지정 가능  
  - `-i [키파일]` : 인증용 개인 키 파일 지정  
  - `-l [사용자명]` : 로그인 사용자 지정  
- **예시:**  
  ```bash
  ssh -p 2222 devuser@192.168.0.10
  ```

---

### 2. SCP (Secure Copy Protocol)
- **역할:** 로컬 ↔ 원격 서버 간 파일 복사  
- **기본 사용법:**  
  ```bash
  scp [옵션] 원본경로 사용자명@호스트주소:목적지경로
  ```
- **주요 옵션:**
  - `-r` : 디렉토리 전체 복사  
  - `-P [포트번호]` : 특정 포트 지정  
  - `-i [키파일]` : SSH 키 파일 사용  
  - `-p` : 원본 파일 권한 유지  
- **예시:**  
  ```bash
  scp -P 2222 todo.txt devuser@myserver.com:/home/devuser/
  scp -r ./project devuser@192.168.0.10:/var/www/
  ```

---

### 3. SFTP (Secure File Transfer Protocol)
- **역할:** SSH 기반의 대화형 파일 전송  
- **기본 사용법:**  
  ```bash
  sftp 사용자명@호스트주소
  ```
- **주요 명령어 (SFTP 세션 내):**
  - `put [파일명]` : 로컬 → 서버 업로드  
  - `get [파일명]` : 서버 → 로컬 다운로드  
  - `ls` : 서버 디렉토리 목록 확인  
  - `cd` : 서버 디렉토리 이동  
- **예시:**  
  ```bash
  sftp devuser@192.168.0.10
  put index.html
  get config.json
  ```

---

## 📊 비교 정리

| 명령어 | 주요 기능 | 사용 상황 | 장점 |
|--------|-----------|-----------|------|
| **SSH** | 원격 서버 접속 및 명령 실행 | 서버 관리, 배포, 로그 확인 | 암호화된 안전한 접속 |
| **SCP** | 파일 복사 | 코드/데이터 전송 | 단순하고 빠른 파일 이동 |
| **SFTP** | 대화형 파일 전송 | 다수 파일 관리, 디렉토리 탐색 | 직관적이고 유연한 파일 관리 |

---

## ⚠️ 주의사항
- **보안:** 항상 암호화된 연결(SSH 기반)을 사용해야 하며, 비밀번호 대신 **SSH 키 인증**을 권장합니다.  
- **덮어쓰기 위험:** SCP는 동일한 파일명일 경우 별도 경고 없이 덮어쓸 수 있으므로 주의해야 합니다.  
- **포트 관리:** 기본 포트(22)를 변경하면 보안성이 높아지지만 접속 시 반드시 `-p` 옵션으로 지정해야 합니다.
