# 서버 프로세스 관리
> **리눅스에서 프로세스를 관리할 때 자주 쓰이는 명령어(ps, top, kill 등)**를 정리
> 서버 운영이나 개발 환경에서 꼭 알아야 하는 기본 도구

---

## 🖥️ 프로세스 관리 기본 명령어

### 1. `ps` (process status)
- **역할:** 현재 실행 중인 프로세스 목록 확인  
- **사용법:**  
  ```bash
  ps [옵션]
  ```
- **주요 옵션:**
  - `ps` : 현재 셸에서 실행 중인 프로세스만 표시  
  - `ps -e` : 시스템 전체 프로세스 표시  
  - `ps -f` : 자세한 정보 표시 (UID, PID, PPID, CMD 등)  
  - `ps aux` : BSD 스타일, CPU/메모리 사용량 포함 전체 프로세스 표시  
- **예시:**  
  ```bash
  ps aux | grep python   # python 관련 프로세스 검색
  ```

---

### 2. `top`
- **역할:** 실시간으로 프로세스 상태 모니터링  
- **사용법:**  
  ```bash
  top
  ```
- **주요 기능:**
  - CPU, 메모리 사용량, 실행 시간, 사용자 등 표시  
  - `k` : 프로세스 종료 (PID 입력 필요)  
  - `q` : 종료  
  - `M` : 메모리 사용량 기준 정렬  
  - `P` : CPU 사용량 기준 정렬  
- **특징:** 서버 자원 모니터링에 필수  

---

### 3. `kill`
- **역할:** 특정 프로세스 종료  
- **사용법:**  
  ```bash
  kill [옵션] PID
  ```
- **주요 옵션/시그널:**
  - `kill -9 PID` : 강제 종료 (SIGKILL)  
  - `kill -15 PID` : 정상 종료 요청 (SIGTERM, 기본값)  
  - `kill -HUP PID` : 프로세스 재시작 요청 (SIGHUP)  
- **예시:**  
  ```bash
  kill -9 1234   # PID 1234 프로세스 강제 종료
  ```

---

### 4. `htop` (추가 도구, 설치 필요)
- **역할:** `top`의 개선판, 인터페이스가 직관적  
- **특징:** 색상 표시, 키보드로 프로세스 선택 후 종료 가능  
- **사용법:**  
  ```bash
  htop
  ```

---

## 📊 비교 정리

| 명령어 | 주요 기능 | 장점 | 단점 |
|--------|-----------|------|------|
| **ps** | 프로세스 목록 출력 | 다양한 옵션으로 상세 확인 가능 | 실시간 모니터링 불가 |
| **top** | 실시간 프로세스 모니터링 | CPU/메모리 사용량 확인 | UI가 단순, 조작 불편 |
| **kill** | 프로세스 종료 | 간단하고 빠름 | PID 알아야 함 |
| **htop** | 실시간 모니터링 + 직관적 UI | 색상/검색/선택 지원 | 기본 설치 안 되어 있을 수 있음 |

---

## ⚠️ 주의사항
- **kill -9**는 프로세스를 강제로 종료하므로 데이터 손실 위험이 있음 → 가급적 `kill -15`로 정상 종료 시도 후 사용  
- **ps aux | grep** 패턴은 특정 프로세스를 찾을 때 가장 많이 쓰임  
- **top/htop**은 서버 자원 모니터링 시 필수적이며, 장시간 실행되는 애플리케이션 관리에 유용
