# 서버 네트워크관리 명령어
> 리눅스에서 네트워크 상태 확인과 통신 테스트에 자주 쓰이는 명령어는 아래와 같다.
- **ifconfig**
- **ping**
- **netstat**
- **curl**
- **wget**

각각 네트워크 인터페이스 확인, 연결 테스트, 포트 상태 확인, 웹 요청, 파일 다운로드에 활용됩니다.

---

## 🌐 네트워크 관련 기본 명령어

### 1. `ifconfig` (interface configuration)
- **역할:** 네트워크 인터페이스 설정 및 상태 확인  
- **사용법:**  
  ```bash
  ifconfig
  ifconfig eth0 192.168.0.10 netmask 255.255.255.0
  ```
- **주요 기능:**
  - IP 주소, MAC 주소 확인  
  - 네트워크 인터페이스 활성화/비활성화 (`up`, `down`)  
- **참고:** 최신 리눅스에서는 `ip addr` 명령어로 대체되는 경우 많음  

---

### 2. `ping`
- **역할:** 네트워크 연결 상태 및 지연 시간 확인  
- **사용법:**  
  ```bash
  ping [옵션] 호스트명/IP
  ```
- **주요 옵션:**
  - `-c [횟수]` : 지정한 횟수만큼 패킷 전송  
  - `-i [간격]` : 패킷 전송 간격 설정  
- **예시:**  
  ```bash
  ping -c 4 google.com
  ```
---

### 3. `netstat`
- **역할:** 네트워크 연결, 포트 상태, 라우팅 테이블 확인  
- **사용법:**  
  ```bash
  netstat [옵션]
  ```
- **주요 옵션:**
  - `-a` : 모든 연결 및 포트 표시  
  - `-t` : TCP 연결만 표시  
  - `-u` : UDP 연결만 표시  
  - `-n` : 호스트명 대신 IP 표시  
- **예시:**  
  ```bash
  netstat -tuln
  ```

---

### 4. `curl`
- **역할:** URL을 통해 데이터 요청 및 응답 확인  
- **사용법:**  
  ```bash
  curl [옵션] URL
  ```
- **주요 기능:**
  - 웹 페이지 내용 가져오기  
  - API 호출 및 응답 확인  
  - 파일 다운로드 (`-O` 옵션)  
- **예시:**  
  ```bash
  curl https://example.com
  curl -X POST -d "id=1&name=test" https://api.example.com/data
  ```

---

### 5. `wget`
- **역할:** 파일 다운로드 전용 명령어  
- **사용법:**  
  ```bash
  wget [옵션] URL
  ```
- **주요 옵션:**
  - `-c` : 중단된 다운로드 이어받기  
  - `-r` : 재귀적으로 전체 사이트 다운로드  
  - `-O [파일명]` : 저장 파일명 지정  
- **예시:**  
  ```bash
  wget https://example.com/file.zip
  wget -c https://example.com/large.iso
  ```

---

## 📊 비교 정리

| 명령어   | 주요 기능 | 사용 예시 | 특징 |
|----------|-----------|-----------|------|
| **ifconfig** | 네트워크 인터페이스 확인/설정 | `ifconfig eth0` | 최신 시스템은 `ip addr` 권장 |
| **ping** | 연결 테스트, 지연 확인 | `ping -c 4 google.com` | 네트워크 문제 진단 필수 |
| **netstat** | 포트/연결 상태 확인 | `netstat -tuln` | 보안 점검 시 유용 |
| **curl** | 웹 요청/응답 확인 | `curl https://api.example.com` | API 테스트에 최적 |
| **wget** | 파일 다운로드 | `wget https://example.com/file.zip` | 대용량 파일 다운로드에 강력 |

---

## ⚠️ 주의사항
- **ifconfig**는 최신 배포판에서 기본 설치되지 않을 수 있으므로 `ip` 명령어 사용을 권장합니다.  
- **netstat**도 `ss` 명령어로 대체되는 경우가 많습니다.  
- **curl/wget** 사용 시 인증서 검증 옵션(`--insecure`)을 남용하면 보안 위험이 있습니다.  
- **ping**은 방화벽 설정에 따라 차단될 수 있으므로 연결 확인이 항상 성공하지는 않습니다.
