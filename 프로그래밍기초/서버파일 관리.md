# 서버파일 관리 명령어
> 리눅스에서 가장 기본적이고 자주 쓰이는 파일/디렉토리 관리 명령어는 **ls, cd, cp, mv, rm**입니다.
> 이들은 디렉토리 탐색, 이동, 복사, 이동/이름 변경, 삭제 작업을 담당하며 Git Bash에서도 동일하게 활용할 수 있습니다.

---

## 📂 자주 쓰이는 파일/디렉토리 관리 명령어

### 1. `ls` (list)
- **역할:** 현재 디렉토리의 파일 및 디렉토리 목록 출력  
- **주요 옵션:**
  - `-l` : 상세 정보(권한, 소유자, 크기, 수정 시간) 표시  
  - `-a` : 숨김 파일(`.`으로 시작하는 파일) 포함  
  - `-h` : 사람이 읽기 쉬운 크기(KB, MB 단위) 표시  
- **예시:**  
  ```bash
  ls -la
  ```

---

### 2. `cd` (change directory)
- **역할:** 디렉토리 이동  
- **사용법:**  
  ```bash
  cd [디렉토리명]
  ```
- **특수 경로:**
  - `cd ..` : 상위 디렉토리로 이동  
  - `cd ~` : 홈 디렉토리로 이동  
  - `cd -` : 직전 디렉토리로 이동  

---

### 3. `cp` (copy)
- **역할:** 파일 또는 디렉토리 복사  
- **주요 옵션:**
  - `-r` : 디렉토리 전체 복사  
  - `-p` : 원본 권한/속성 유지  
  - `-i` : 덮어쓰기 전에 확인  
- **예시:**  
  ```bash
  cp file.txt backup.txt
  cp -r project/ /home/user/
  ```

---

### 4. `mv` (move)
- **역할:** 파일 이동 또는 이름 변경  
- **사용법:**  
  ```bash
  mv 원본경로 대상경로
  ```
- **예시:**  
  ```bash
  mv report.txt /home/user/docs/
  mv oldname.txt newname.txt
  ```

---

### 5. `rm` (remove)
- **역할:** 파일/디렉토리 삭제  
- **주요 옵션:**
  - `-r` : 디렉토리와 그 안의 모든 파일 삭제  
  - `-f` : 강제 삭제(확인 없이)  
  - `-i` : 삭제 전 확인  
- **예시:**  
  ```bash
  rm file.txt
  rm -rf temp/
  ```

---

## 📊 비교 정리

| 명령어 | 주요 기능 | 자주 쓰이는 옵션 | 주의사항 |
|--------|-----------|------------------|----------|
| **ls** | 파일/디렉토리 목록 확인 | `-l`, `-a`, `-h` | 숨김 파일 확인 시 `-a` 필요 |
| **cd** | 디렉토리 이동 | `..`, `~`, `-` | 잘못된 경로 입력 시 오류 발생 |
| **cp** | 파일/디렉토리 복사 | `-r`, `-p`, `-i` | 덮어쓰기 위험, `-i` 권장 |
| **mv** | 파일 이동/이름 변경 | 없음 | 덮어쓰기 시 원본 손실 가능 |
| **rm** | 파일/디렉토리 삭제 | `-r`, `-f`, `-i` | 삭제 후 복구 불가, 신중히 사용 |

---

## ⚠️ 주의사항
- **rm -rf**는 매우 위험합니다. 잘못 입력하면 시스템 전체 파일이 삭제될 수 있으므로 항상 경로를 확인해야 합니다.  
- **cp/mv**는 덮어쓰기 시 별도 경고 없이 진행될 수 있으므로 `-i` 옵션을 습관적으로 사용하는 것이 안전합니다.  
- Git Bash에서도 동일하게 동작하지만, Windows 경로(`C:/Users/...`)와 혼용 시 주의가 필요합니다.

## 📖 파일 내용 확인 명령어

### 1. `cat` (concatenate)
- **역할:** 파일 내용을 한 번에 출력  
- **사용법:**  
  ```bash
  cat filename.txt
  ```
- **주요 활용:**
  - 여러 파일을 이어서 출력: `cat file1.txt file2.txt`  
  - 파일 생성: `cat > newfile.txt` (입력 후 `Ctrl+D`로 종료)  
- **특징:** 작은 파일 확인에 적합, 대용량 파일은 불편  

---

### 2. `less`
- **역할:** 대용량 파일을 페이지 단위로 확인  
- **사용법:**  
  ```bash
  less filename.txt
  ```
- **주요 조작 키:**
  - `Space` : 다음 페이지  
  - `b` : 이전 페이지  
  - `/검색어` : 특정 문자열 검색  
  - `q` : 종료  
- **특징:** 로그 파일이나 긴 텍스트 확인에 최적  

---

## ✍️ 파일 편집 명령어

### 3. `nano`
- **역할:** 간단하고 직관적인 텍스트 편집기  
- **사용법:**  
  ```bash
  nano filename.txt
  ```
- **주요 단축키:**
  - `Ctrl+O` : 저장  
  - `Ctrl+X` : 종료  
  - `Ctrl+K` : 한 줄 잘라내기  
  - `Ctrl+U` : 붙여넣기  
- **특징:** 초보자 친화적, 빠른 수정에 적합  

---

### 4. `vi` (또는 `vim`)
- **역할:** 강력한 텍스트 편집기, 개발자들이 많이 사용  
- **사용법:**  
  ```bash
  vi filename.txt
  ```
- **모드 구조:**
  - **명령 모드** : 기본 모드, 커서 이동/삭제/복사 등  
  - **입력 모드** : `i` 키로 진입, 실제 텍스트 입력  
  - **명령 실행 모드** : `:` 입력 후 저장/종료 등 실행  
- **주요 명령어:**
  - `i` : 입력 모드 진입  
  - `:w` : 저장  
  - `:q` : 종료  
  - `:wq` : 저장 후 종료  
  - `dd` : 현재 줄 삭제  
  - `yy` : 현재 줄 복사  
  - `p` : 붙여넣기  
- **특징:** 러닝 커브가 있지만 강력한 기능 제공  

---

## 📊 비교 정리

| 명령어 | 주요 기능 | 장점 | 단점 |
|--------|-----------|------|------|
| **cat** | 파일 전체 출력 | 간단, 빠름 | 대용량 파일 불편 |
| **less** | 페이지 단위 출력 | 검색/스크롤 가능 | 편집 불가 |
| **nano** | 직관적 편집 | 쉬운 사용법 | 고급 기능 부족 |
| **vi/vim** | 강력한 편집 | 다양한 기능, 개발자 필수 | 배우기 어려움 |

---

## ⚠️ 팁
- **로그 확인** → `less` 추천  
- **빠른 수정** → `nano` 추천  
- **코드 편집/고급 작업** → `vi/vim` 필수  
- Git Bash에서는 기본적으로 `cat`, `less`, `nano`, `vi` 모두 사용 가능하지만, Windows 환경에서는 `nano`와 `vi`가 설치되어 있지 않을 수 있으니 확인 필요

## 🔐 파일 권한 및 소유권 관리 명령어

### 1. `chmod` (change mode)
- **역할:** 파일/디렉토리의 접근 권한 변경  
- **사용법:**  
  ```bash
  chmod [옵션] 권한 파일명
  ```
- **권한 지정 방식:**
  - **기호 방식:**  
    - `u` (사용자), `g` (그룹), `o` (기타), `a` (전체)  
    - `+` (권한 추가), `-` (권한 제거), `=` (권한 설정)  
    ```bash
    chmod u+x script.sh   # 사용자에게 실행 권한 추가
    chmod g-w report.txt  # 그룹의 쓰기 권한 제거
    ```
  - **숫자 방식:**  
    - `r=4`, `w=2`, `x=1` → 합산하여 권한 지정  
    - 예: `chmod 755 file.txt` → 사용자(rwx=7), 그룹(rx=5), 기타(rx=5)  
    ```bash
    chmod 644 document.txt
    ```

---

### 2. `chown` (change owner)
- **역할:** 파일/디렉토리의 소유자(owner)와 그룹(group) 변경  
- **사용법:**  
  ```bash
  chown [옵션] 사용자[:그룹] 파일명
  ```
- **예시:**
  ```bash
  chown alice file.txt          # 소유자를 alice로 변경
  chown alice:dev file.txt      # 소유자 alice, 그룹 dev로 변경
  chown -R bob project/         # 디렉토리 전체를 bob 소유로 변경
  ```

---

### 3. `chgrp` (change group)
- **역할:** 파일/디렉토리의 그룹만 변경  
- **사용법:**  
  ```bash
  chgrp [옵션] 그룹명 파일명
  ```
- **예시:**
  ```bash
  chgrp developers file.txt
  chgrp -R staff /var/www/
  ```

---

## 📊 비교 정리

| 명령어   | 주요 기능 | 사용 예시 | 특징 |
|----------|-----------|-----------|------|
| **chmod** | 권한 변경 | `chmod 755 file.txt` | 숫자/기호 방식 모두 지원 |
| **chown** | 소유자/그룹 변경 | `chown alice:dev file.txt` | 소유자와 그룹 동시 변경 가능 |
| **chgrp** | 그룹 변경 | `chgrp staff file.txt` | 그룹만 변경할 때 사용 |

---

## ⚠️ 주의사항
- **권한 설정 실수**는 보안 취약점으로 이어질 수 있음 → 최소 권한 원칙 적용  
- **chown/chgrp**는 일반 사용자 권한으로는 제한적, 보통 **root 권한** 필요  
- **chmod 777**은 모든 사용자에게 모든 권한을 주는 위험한 설정 → 피하는 것이 좋음
