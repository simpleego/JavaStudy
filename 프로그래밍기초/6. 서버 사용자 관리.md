# 👤서버 사용자 관리 명령어   
> 리눅스 서버에서 사용 관리를 위한 사용자 계정과 권한을 관리할 때 자주 쓰이는 명령어
- **useradd**
- **passwd**
- **su**
- **sudo**
---

## 👥 사용자 계정 및 권한 관리 명령어

### 1. `useradd`
- **역할:** 새로운 사용자 계정 생성  
- **사용법:**  
  ```bash
  useradd [옵션] 사용자명
  ```
- **주요 옵션:**
  - `-m` : 홈 디렉토리 자동 생성  
  - `-d [경로]` : 홈 디렉토리 지정  
  - `-s [쉘]` : 로그인 시 사용할 쉘 지정  
- **예시:**  
  ```bash
  useradd -m -s /bin/bash alice
  ```

---

### 2. `passwd`
- **역할:** 사용자 계정의 비밀번호 설정/변경  
- **사용법:**  
  ```bash
  passwd [사용자명]
  ```
- **예시:**  
  ```bash
  passwd alice   # alice 계정 비밀번호 설정
  passwd         # 현재 로그인한 사용자 비밀번호 변경
  ```

---

### 3. `su` (substitute user)
- **역할:** 다른 사용자 계정으로 전환  
- **사용법:**  
  ```bash
  su [옵션] [사용자명]
  ```
- **주요 옵션:**
  - `su` : root 계정으로 전환 (비밀번호 필요)  
  - `su -` : 환경 변수까지 포함해 완전한 로그인 쉘로 전환  
  - `su alice` : alice 계정으로 전환  
- **예시:**  
  ```bash
  su - root
  su - alice
  ```

---

### 4. `sudo` (superuser do)
- **역할:** 특정 명령어를 관리자 권한으로 실행  
- **사용법:**  
  ```bash
  sudo [명령어]
  ```
- **주요 기능:**
  - root 권한 없이도 특정 명령 실행 가능  
  - `/etc/sudoers` 파일에서 권한 제어 가능  
- **예시:**  
  ```bash
  sudo apt update
  sudo systemctl restart nginx
  ```

---

## 📊 비교 정리

| 명령어   | 주요 기능 | 사용 예시 | 특징 |
|----------|-----------|-----------|------|
| **useradd** | 사용자 계정 생성 | `useradd -m alice` | 홈 디렉토리/쉘 지정 가능 |
| **passwd** | 비밀번호 설정/변경 | `passwd alice` | 보안 관리 필수 |
| **su** | 다른 사용자로 전환 | `su - root` | root 권한 필요 |
| **sudo** | 관리자 권한으로 명령 실행 | `sudo apt update` | sudoers 설정으로 제어 가능 |

---

## ⚠️ 주의사항
- **root 계정 직접 사용**은 위험할 수 있으므로 `sudo` 사용을 권장합니다.  
- **비밀번호 관리**는 보안의 핵심 → 정기적으로 변경하고 복잡하게 설정해야 합니다.  
- **sudoers 파일 수정** 시 `visudo` 명령을 사용해야 문법 오류로 인한 시스템 잠금 위험을 줄일 수 있습니다.
