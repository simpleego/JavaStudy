## 💻 웹 개발자 과정 필수 명령줄 명령어 (CLI) 3단계 정리

웹 개발 과정에서 효율적인 작업 환경을 구축하고 서버, Git, 빌드 시스템 등을 다루기 위해 명령줄(CLI) 명령어는 필수적

---

### 1단계: 📁 기초적인 파일 및 디렉터리 관리 (Fundamentals)

가장 기본적인 파일 시스템 탐색 및 관리를 위한 명령어입니다. 운영체제(Windows, macOS, Linux)에 관계없이 자주 사용됩니다.

| 명령어 | 용도 | 예시 |
| :--- | :--- | :--- |
| **`pwd`** | 현재 작업 중인 디렉터리의 전체 경로를 출력합니다. (Print Working Directory) | `pwd` |
| **`ls`** | 현재 디렉터리 안의 파일과 폴더 목록을 출력합니다. (List) | `ls -al` (숨김 파일 포함 상세 정보 출력) |
| **`cd`** | 디렉터리(폴더)를 이동합니다. (Change Directory) | `cd project` (project 폴더로 이동), `cd ..` (상위 폴더로 이동) |
| **`mkdir`** | 새로운 디렉터리(폴더)를 생성합니다. (Make Directory) | `mkdir components` |
| **`touch`** | 비어 있는 새 파일을 생성하거나, 기존 파일의 접근 시간을 업데이트합니다. | `touch index.html` |
| **`cat`** | 파일 내용을 명령창에 출력합니다. (Concatenate) | `cat script.js` |
| **`cp`** | 파일이나 디렉터리를 복사합니다. (Copy) | `cp file.txt backup/` |
| **`mv`** | 파일이나 디렉터리를 이동하거나 이름을 변경합니다. (Move) | `mv old.js new.js` |
| **`rm`** | 파일이나 디렉터리를 삭제합니다. (Remove) | `rm temp.txt` |
| **`clear`** | 명령창의 내용을 지우고 깨끗하게 만듭니다. | `clear` |

---

### 2단계: ⚙️ 고급 파일 관리 및 검색 (Advanced File Handling)

파일 내용을 검색하거나, 복잡한 경로에서 파일을 다루는 등 개발 환경에서 유용한 명령어입니다.

| 명령어 | 용도 | 예시 |
| :--- | :--- | :--- |
| **`grep`** | 파일 내용에서 특정 패턴이나 문자열을 검색합니다. | `grep "function" app.js` |
| **`find`** | 지정한 경로에서 특정 이름이나 속성을 가진 파일을 검색합니다. | `find . -name "*.js"` (현재 폴더에서 모든 js 파일 검색) |
| **`tar`** | 여러 파일을 묶거나 압축/압축 해제합니다. (주로 서버에서 사용) | `tar -czvf archive.tar.gz myfolder/` |
| **`chmod`** | 파일 및 디렉터리의 접근 권한을 변경합니다. (Change Mode) | `chmod 755 script.sh` (스크립트 실행 권한 부여) |
| **`chown`** | 파일 및 디렉터리의 소유자나 그룹을 변경합니다. (Change Owner) | `chown user:group file.txt` |
| **`diff`** | 두 파일의 내용을 비교하여 차이점을 출력합니다. (Difference) | `diff file1.txt file2.txt` |
| **`alias`** | 길거나 복잡한 명령에 짧은 별칭을 지정합니다. (Bash/Zsh 등 셸 환경 설정 시 유용) | `alias gs='git status'` |

### 3단계: 🌐 네트워크 및 프로세스 관리 (Network & Process)

로컬 개발 서버 실행, 외부 서버 접속, 포트 확인 등 실제 애플리케이션 운영 환경과 관련된 명령어입니다.

| 명령어 | 용도 | 예시 |
| :--- | :--- | :--- |
| **`ping`** | 특정 네트워크 주소(IP 또는 도메인)에 도달 가능한지 확인합니다. (네트워크 연결 테스트) | `ping google.com` |
| **`curl`** | URL을 사용하여 데이터를 전송하거나 수신합니다. (API 테스트, 외부 데이터 요청) | `curl -X POST http://api.example.com/data` |
| **`ssh`** | 원격 서버에 안전하게 접속합니다. (Secure Shell) | `ssh user@remote-server.com` |
| **`ps`** | 현재 실행 중인 프로세스 목록을 확인합니다. (Process Status) | `ps aux` |
| **`kill`** | 특정 프로세스 ID(PID)를 사용하여 프로세스를 종료합니다. | `kill 12345` |
| **`netstat`** | 현재 시스템의 네트워크 연결 상태, 라우팅 테이블, 포트 사용 현황 등을 확인합니다. | `netstat -an` |
| **`top`** / **`htop`** | 시스템의 실시간 프로세스 사용 현황(CPU, 메모리)을 모니터링합니다. | `top` |
| **`npm`** / **`yarn`** | (Node.js 환경) 패키지 설치 및 프로젝트 빌드 명령을 실행합니다. | `npm install` / `npm run build` |
| **`git`** | 소스 코드 버전 관리를 합니다. (웹 개발자에게 가장 중요) | `git clone` / `git push` / `git commit` |
