# 🧩 **1. 추상클래스(Abstract Class)란?**

추상클래스는 **객체를 직접 만들 수 없는 미완성 클래스**야.  
즉, “공통적인 틀은 제공하지만, 구체적인 동작은 자식이 완성해야 하는 클래스”.

### ✔ 핵심 특징
- `abstract` 키워드로 선언
- **추상 메서드(abstract method)** 를 포함할 수 있음  
  → 몸체가 없는 메서드  
- **일반 메서드도 포함 가능**  
  → 인터페이스와의 큰 차이
- **필드, 생성자 모두 가질 수 있음**
- 반드시 **상속을 통해서만 사용**됨

---

# 🎯 **2. 추상클래스를 사용하는 목적**

### ✔ (1) 공통된 규칙(템플릿)을 강제하기 위해  
예: 모든 동물은 반드시 “울다” 기능을 가져야 한다.

### ✔ (2) 상속 구조에서 공통 기능은 제공하고, 다른 기능은 자식이 구현하도록 하기 위해  
예: 이동 방식은 다르지만 “이동한다”는 공통 개념은 유지.

### ✔ (3) 객체를 직접 만들면 안 되는 상위 개념을 표현하기 위해  
예: “동물”, “탈것”, “직원” 같은 개념은 실제 객체가 될 수 없음.

# 🚗 **3. 추상클래스를 가장 적절하게 사용하는 예제**

## 🎬 예제 주제: **탈것(Vehicle) 추상클래스**

탈것은 “움직인다”라는 공통 개념을 갖지만  
- 자동차는 바퀴로 움직이고  
- 배는 물 위를 떠서 움직이고  
- 비행기는 날아서 움직이지

즉, **공통 개념은 같지만 구현 방식은 다름** → 추상클래스가 딱 맞는 상황.

# 🧱 **4. 추상클래스 예제 코드**

## 🔹 추상클래스 Vehicle

```java
abstract class Vehicle {
    private String name;

    public Vehicle(String name) {
        this.name = name;
    }

    // 공통 기능
    public void start() {
        System.out.println(name + " 시동을 켭니다.");
    }

    public void stop() {
        System.out.println(name + " 시동을 끕니다.");
    }

    // 자식이 반드시 구현해야 하는 기능
    public abstract void move();

    public String getName() {
        return name;
    }
}
```

---

## 🔹 자동차 Car (Vehicle 상속)

```java
class Car extends Vehicle {
    public Car(String name) {
        super(name);
    }

    @Override
    public void move() {
        System.out.println(getName() + "이(가) 도로를 달립니다.");
    }
}
```

---

## 🔹 배 Ship

```java
class Ship extends Vehicle {
    public Ship(String name) {
        super(name);
    }

    @Override
    public void move() {
        System.out.println(getName() + "이(가) 바다를 항해합니다.");
    }
}
```

---

## 🔹 비행기 Airplane

```java
class Airplane extends Vehicle {
    public Airplane(String name) {
        super(name);
    }

    @Override
    public void move() {
        System.out.println(getName() + "이(가) 하늘을 납니다.");
    }
}
```

---

## 🔹 테스트 코드

```java
public class VehicleTest {
    public static void main(String[] args) {
        Vehicle v1 = new Car("자동차");
        Vehicle v2 = new Ship("배");
        Vehicle v3 = new Airplane("비행기");

        v1.start();
        v1.move();
        v1.stop();

        System.out.println();

        v2.start();
        v2.move();
        v2.stop();

        System.out.println();

        v3.start();
        v3.move();
        v3.stop();
    }
}
```

---

# 🎯 **5. 이 예제가 추상클래스에 가장 적절한 이유**

| 이유 | 설명 |
|------|------|
| **공통 기능 존재** | start(), stop()은 모든 탈것이 동일 |
| **구현이 달라야 하는 기능 존재** | move()는 탈것마다 다름 → 추상 메서드 적합 |
| **상위 개념은 객체가 될 수 없음** | Vehicle 자체는 실체가 아님 |
| **다형성 활용 가능** | Vehicle 타입으로 Car/Ship/Airplane 모두 다룸 |

---

# 🎉 ** 핵심 메시지 **

- 추상클래스는 “공통된 틀을 제공하는 미완성 설계도”
- 객체를 만들 수 없고, 반드시 상속해서 사용
- 공통 기능은 부모가 제공하고, 다른 기능은 자식이 구현
- 다형성과 함께 사용할 때 진가가 드러남
